For Linux, the default depends on how Docker is installed.

For the normal rootful Docker daemon, a named volume like `mongo-data` lives under:

```bash
/var/lib/docker/volumes/<volume-name>/_data
```

In your compose project, the effective volume name is usually:

```bash
aichatbotjava_mongo-data
```

so the mountpoint is typically:

```bash
/var/lib/docker/volumes/aichatbotjava_mongo-data/_data
```

If you are using rootless Docker, the path is usually under your home directory, for example:

```bash
~/.local/share/docker/volumes/<volume-name>/_data
```

If you are using Docker Desktop on Linux, the volume still exists inside Docker’s managed storage, but the exact backing path may differ from the plain `/var/lib/docker` layout.

To check the exact path on your machine:

```bash
docker volume inspect aichatbotjava_mongo-data
```
